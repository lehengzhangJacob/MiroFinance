\begin{abstract}
\miroflow{} 具备长链推理与多工具编排能力，但任务结束后上下文随即销毁，历史交易经验和可复用决策流程不能跨任务沉淀。本文在其上实现 \memskill{}：将到期后才可见的事实型组合记忆、带触发条件的程序性技能以及点时金融工具统一接入推理过程，并进一步构建受静态门禁、时间切分和封存集约束的文本技能自进化闭环。为避免固定股票池和分类标签偏置，我们设计了全 A 股开放池月度组合基准：智能体每月从约 4,800 只点时可交易股票中配置组合，持有 20 个交易日；评估器执行 100 股整手、真实费率、显式现金与顺序复利。在 2024 年 7 月至 2025 年 6 月的 12 个月正式对照中，\miroflow{} 与 \memskill{} 的累计净收益分别为 $21.28\%$ 和 $38.18\%$，月频年化 Sharpe 分别为 0.78 和 1.12；后者提高 $16.90\pp$ 收益，但最大回撤由 $-14.81\%$ 扩大至 $-17.01\%$。在独立的 24 个月技能进化实验中，第一轮晋升技能在开发集和封存集上分别取得 $+6.78\pp$ 与 $+5.94\pp$ 的月均配对提升，均为 5 胜 1 负并通过硬门禁。结果表明，外部长期认知层和可审计技能优化能够改善单次金融智能体决策，但小样本、单次随机 rollout 与 API 稳定性使结论仍属于机制验证，而非可交易性证明。
\end{abstract}

\section{引言}
\label{sec:intro}

工具型大模型智能体已经能够检索资料、运行代码并在较长轨迹中组织证据。\miroflow{}~\citep{miroflow2025} 是其中一个典型框架：主智能体通过 MCP 协议调用外部工具，也可以把子任务委托给专业子智能体。然而，其默认认知状态仍主要存在于单次上下文中。任务完成后，哪些筛选步骤有效、哪些风险被忽略、怎样检查最终格式等经验不会自动进入下一次任务。对于每月重复决策的金融场景，这意味着智能体持续支付相同的探索成本，也可能反复犯同类错误。

从系统原理看，\miroflow{} 采用中心 orchestrator 保留推理控制、LLM client 生成动作、MCP tool manager 执行外部操作，并把子智能体也抽象成可调用工具。这种``中心推理、外围能力''的边界意味着改进不必重写主循环：更合适的切入点是在任务开始前注入可用知识、在推理中提供按需检索、在任务结算后写入可验证经验。本文据此把 memory 与 skill 设计为原编排器之外的长期认知层，而不是不可审计地扩充模型上下文。

实践考核要求包含三个层次：重构 \miroflow{} 的认知逻辑以联动工具、memory 与 skill；迁移到 A 股资产定价任务并做定量、定性验证；继续通过自进化或多智能体提升效果。本文围绕这三个目标完成如下工作：
\begin{enumerate}
    \item 实现一个与原编排器解耦的长期认知层，将事实型情景记忆和程序性技能同时注入主智能体，并保留推理期按需加载接口；
    \item 构建全 A 股开放池、按 \texttt{as\_of} 截断的组合任务和确定性回放器，把整手、费用、未成交资金及无效输出纳入评价；
    \item 借鉴 Hermes Agent Self-Evolution~\citep{hermes2025}，实现技能文本的 train--dev--holdout 演化、差分静态门禁、配对收益评分、一次性封存集租约和可回滚晋升；
    \item 落地基于 \miroflow{} 原生子智能体接口的筛选员--分析员分工，并如实记录尚未完成的正式架构对比、失败尝试和效度边界。
\end{enumerate}

本文的主结论来自两个互补实验。第一组直接比较工具相同的 \miroflow{} 与 ``memory+skill'' 联合系统；第二组关闭运行时记忆，只改变注入的单份技能文本，从而隔离技能进化的增量。所有收益数字均来自冻结数据库上的确定性回放，不使用 LLM 裁判。

\section{相关工作}
\label{sec:related}

\paragraph{\miroflow{}：工具增强的深度研究框架。}
\miroflow{} 是面向复杂研究与预测任务的模块化智能体框架，支持多轮工具推理、MCP 工具生态和层次化子智能体编排~\citep{miroflow2025}。它为``模型生成动作--工具返回观察--主智能体继续推理''提供了可复用执行骨架，公开设计的重点则是单次任务中的长轨迹完成、并发与可靠性，而非跨任务知识的成熟、检索和治理。本文保留其 orchestrator、工具管理器和原生子智能体接口，只在任务边界外增加可审计的长期认知层；因此研究问题不是重新设计通用编排器，而是使原有执行框架能够安全复用历史经验。

\paragraph{Qlib：AI 驱动的量化研究平台。}
Qlib 将市场数据处理、特征与模型训练、组合策略和回测组织为统一的量化研究工作流，强调实验效率与可复现性~\citep{qlib2020}。它主要研究从数据到信号、策略和回测的系统化管线，而本文研究的是 LLM 智能体如何调用点时工具、使用跨任务记忆并生成可执行组合。本文没有把 Qlib 的特定预测模型作为主对照，但沿用其将数据、决策与评价解耦的思想，并进一步针对开放池智能体加入 \texttt{as\_of} 截断、整手成交、显式现金、交易费用和无效输出处理。

\paragraph{\hermes{}：文本级智能体自进化。}
Hermes Agent Self-Evolution 使用 DSPy 与 GEPA，根据执行轨迹进行反思式搜索，生成并评价 skill 文本候选，再以测试、大小和语义保持等约束筛选变体~\citep{hermes2025}。其总体路线还覆盖工具描述、系统提示和代码优化，而公开实现首先聚焦技能文件。本文借鉴``轨迹反馈--文本变异--受约束选择''这一思想，但没有直接复现其通用优化器：我们将评价替换为冻结 A 股数据上的确定性回放，并加入按时间排序的 train--dev--holdout、一次性封存集租约和可回滚晋升。由此，\miroflow{}、Qlib 与 \hermes{} 分别提供执行框架、金融评测方法论和技能优化范式；本文的贡献在于把三者连接为具有时间防泄漏与部署门禁的金融智能体闭环。

\section{方法}
\label{sec:methods}

\subsection{系统架构与认知机制}
\label{sec:system}

\subsubsection{\miroflow{} 的执行路径与缺口}

\miroflow{} 的主循环由 orchestrator、LLM client 和 MCP tool manager 组成。任务题面进入主智能体后，模型在多轮 ``思考--工具调用--观察'' 中收集证据，最终输出答案；若配置了 \texttt{sub\_agents}，子智能体被包装为可调用工具，在独立会话中执行子任务并返回摘要。这一结构适合单次深度研究，却没有规定跨任务状态的写入、检索、成熟时间和冲突处理。简单地把历史轨迹拼回提示词还会引入三个问题：上下文膨胀、错误叙事被当作规则，以及未来标签泄漏到过去任务。

\subsubsection{工具--记忆--技能三路径联动}

图~\ref{fig:architecture} 给出最终系统。长期认知层不替换原 orchestrator，而是通过三个稳定接口接入：任务前构造检索块；推理中暴露技能加载工具；任务结束后由确定性评估结果写入事实型 episode。这样保留了 \miroflow{} 原工具生态，同时使认知状态可检索、可审计和可按时间过滤。

\subsubsection{事实型情景记忆}
\label{sec:memory}

早期版本参考 Reflexion~\citep{shinn2023reflexion} 和 Mem0~\citep{mem0}，尝试让 LLM 从正误反馈中生成自然语言教训。但小样本金融反馈容易形成方向先验：同一段失败叙事可能被重复写入，语义检索又会放大看多或看空偏置。因此正式开放池实验采用更保守的设计：
\begin{itemize}
    \item \textbf{确定性写入。} 每期保存目标权重、现金、按比例仓位计算的毛/净收益、指数收益、费用以及持仓贡献，不保存模型自由叙事；写入使用 \texttt{infer=False}，避免 LLM 改写事实。该 episode 归因不做 100 股整手舍入，最终报告的回放器才执行整手约束。
    \item \textbf{退出日禁运。} 每条记录携带精确的 \texttt{available\_after}。只有退出日不晚于新任务 \texttt{as\_of} 的 episode 才能被读取；缺少日期的记录默认隐藏。
    \item \textbf{有限窗口。} 正式配置最多注入 3 条到期 episode，并关闭通用语义教训注入（\texttt{inject\_top\_k=0}），避免历史股票名单变成当前推荐。
    \item \textbf{逐月屏障。} 任务按日期顺序、单并发执行，同月完成后再更新状态。即使恢复运行或重排文件，时间过滤仍构成第二道防线。
\end{itemize}

记忆的作用被限定为校准：提醒智能体此前的现金水平、集中度、执行偏差和已实现回撤；当前股票仍必须由当前时点的工具证据重新筛选。这一设计牺牲了开放式语言归纳能力，但显著收窄了泄漏和错误累积面。

\subsubsection{程序性技能}
\label{sec:skill}

技能采用带 YAML frontmatter 的 Markdown 文件组织。frontmatter 描述名称、适用任务、依赖工具和触发词；正文描述可执行流程，而不是固定股票答案。路由器根据任务压缩查询匹配 Top-$k$ 技能，正式实验取 Top-1，并在首轮提示被动给出技能预览。框架代码支持按需 \texttt{skill\_load}，但本次开放池正式对照没有把主动 memory/skill 工具作为实验变量。

开放池技能 \texttt{ashare\_open\_portfolio} 将组合构建拆为七步：确认市场与约束、多路全市场筛选、成长质量交叉验证、先排除后排序、行业分散、动态现金和最终格式复核。技能明确禁止继承旧股票名单，要求所有结论重新满足 \texttt{as\_of}。与情景记忆相比，技能表示``怎样做''，episode 表示``过去实际发生了什么''；两者分别对应程序性知识和情景知识。

\subsubsection{多智能体扩展}
\label{sec:multiagent}

多智能体版本复用原生 \texttt{sub\_agents} 接口，而非另建编排器。主智能体只保留市场状态工具并负责最终组合；\texttt{agent-screener} 承担多路全市场筛选和价格/估值初查，\texttt{agent-analyst} 承担候选股的成长质量、财务与估值批量核验。主智能体通过 \texttt{execute\_subtask} 接收结构化摘要。该分工减少大表直接进入主上下文，但每月需要多段独立 LLM 轨迹，成本和限频风险明显高于单智能体。正式 24 个月运行在本文结果冻结时仍未完成，因此只报告设计，不把中途收益当作架构结论。

\subsection{全 A 股开放池评测}
\label{sec:benchmark}

\subsubsection{任务形式化}

对每个决策日 $t$，点时可交易股票集合记为 $\mathcal{U}_t$。智能体从约 4,800 只当日有成交、非 ST、未退市股票中选择权重 $w_{i,t}$，并显式给出现金权重 $w^{\mathrm{cash}}_t$：
\begin{equation}
  \sum_{i\in\mathcal{U}_t} w_{i,t} + w^{\mathrm{cash}}_t = 1,\qquad
  0\leq w_{i,t}\leq 0.25.
\end{equation}
系统建议持有 4--10 只股票，仅做多、无杠杆。组合在 $t$ 日收盘买入，持有 $H=20$ 个交易日后全部按收盘卖出。初始资本为 100 万元，月间顺序复利：
\begin{equation}
  C_{t+1}=C_t\left(1+R^{\mathrm{net}}_t\right).
\end{equation}
模型最后一行必须输出
\texttt{\textbackslash boxed\{代码:权重,\ldots,CASH:权重\}}。
解析失败、权重和不为 1、缺少 \cash{} 或单股超限均视为无效，本月执行 100\% 现金。

主 memory/skill 对照覆盖 2024-07 至 2025-06 共 12 个月。Hermes 自进化使用扩展至 2026-06 的冻结 24 月快照，按时间顺序划分 train/dev/holdout 为 12/6/6 个月。与最初的 16 股方向分类任务相比，开放池任务不再用窗口内实现收益挑选固定股票，也直接评价可执行组合，因此被选为最终主基准。

\subsubsection{确定性回放、交易摩擦与评价指标}
\label{sec:replay}

评价器将目标权重映射为 100 股整数手。买入费率为 0.05\%，卖出费率为 0.15\%，每笔单边最低 5 元；目标金额不足一手或扣费后不可成交的股票留作现金。持有期收益由 SQLite 日涨跌幅顺序复合，停牌期间没有虚构价格，窗口末按最后可交易状态计值。所有仓位在月末清算，再将净资产带入下一月。

这一回放比简单的权重乘收益更严格。高价股可能因整手约束完全未成交；分散组合会支付更多最低费用；模型写出的现金与实际剩余现金也可能不同。盈利能力以累计净收益率为主：
\begin{equation}
  R_{\mathrm{cum}}=\prod_{t=1}^{T}\left(1+R^{\mathrm{net}}_t\right)-1.
\end{equation}
风险调整收益采用月度净收益的年化 Sharpe：
\begin{equation}
  \mathrm{Sharpe}_{\mathrm{ann}}
  =\sqrt{12}\,
  \frac{\overline{R^{\mathrm{net}}}-r_{f,m}}
       {s\!\left(R^{\mathrm{net}}\right)}.
\end{equation}
其中 $s(\cdot)$ 为样本标准差。为避免引入与回测日期未严格同步的外部利率序列，正式比较统一取月度无风险利率 $r_{f,m}=0$；因此该值用于同窗策略间比较，不解释为绝对投资绩效。最大回撤由每个 20 交易日持有期结束后的净值序列 $V_t$ 计算：
\begin{equation}
  \mathrm{MDD}
  =\min_t\left(\frac{V_t}{\max_{0\leq s\leq t}V_s}-1\right).
\end{equation}
它是月末路径回撤而非持有期内日频回撤。最终统一汇报累计净收益、月频年化 Sharpe、相对基准收益、最大回撤、最差月、正收益月份比例、总费用和有效月数；避免使用含义不清的``盈利率''统称收益率与胜率。

\subsubsection{分层基线}
\label{sec:baselines}

为区分市场贝塔、简单因子和智能体推理，主实验包含以下基线：
\begin{itemize}
    \item 沪深 300 指数，以及 90\% 沪深 300/中证 500/中证 1000 ETF 等权核心加 10\% 现金的可交易基线；
    \item 全市场 20 日相对动量 Top-4、低 PE Top-4 和缓存候选成长规则；
    \item 100 个固定随机种子的流动性股票组合分布；
    \item 全市场全等权风格参考。该项只计比例费用，不执行约 4,800 笔整手与最低费，故明确标注为不可交易上界；
    \item 配置匹配的 \miroflow{}-Plain workspace fork：使用相同 GLM、市场工具和任务顺序。实验 fork 主要增加长期认知层与技能，但也包含更严格的股票池/\texttt{as\_of} 检查和组合修复路径，因此不是同一代码库中的严格单变量消融。
\end{itemize}

\subsubsection{Hermes 技能进化协议}
\label{sec:evolution}

自进化不更新模型参数，只演化一份 Markdown 技能。流程如下：
\begin{enumerate}
    \item 在 12 个 train 月运行当前技能，根据已结算组合轨迹构造反馈；
    \item GLM 反射式生成 3 个候选。L0 静态门禁要求名称和 frontmatter 不变、正文非空且长度受限，并禁止新增具体股票代码和日期；
    \item 当前技能与每个候选在相同 6 个 dev 月独立运行。候选排序分数为月均配对收益差，减去最大回撤恶化的 $0.25$ 倍惩罚；
    \item 只让 dev 最优且通过门禁的候选进入一次性 holdout。任何无效月直接否决，最大回撤相对基线恶化不得超过 $5\pp$；
    \item holdout 通过后才允许晋升，生产技能先备份；registry 保存 digest、parent、报告、租约和回滚记录。
\end{enumerate}

Hermes arm 关闭语义记忆、episode、反思和结果日志，候选与基线仅在技能正文上不同；数据库、任务、模型与评估器保持一致。这使第二组实验能够比主对照更干净地归因于程序性技能文本。

\section{讨论}
\label{sec:discussion}

\subsection{协议修正：从固定池分类到开放池组合}

项目最初采用 16 只股票、12 个月、共 192 个``跑赢/跑输''方向任务。该基准便于确定性判分，却暴露了一个方法学陷阱：为平衡标签而根据窗口内已实现收益补入强势股票，会把未来表现用于构造股票池。即使单个任务的数据仍按 \texttt{as\_of} 截断，整体 benchmark 已不再代表事前可投资域。

因此最终实验转向全 A 股开放池，不再事后挑选任务股票，并把分类正确率替换为可执行组合净收益。固定池实验保留为协议迭代记录，不作为最终效果证据。这一转向也改变了 memory 的单位：从``某只股票方向是否判断正确''改为``一期组合在成本后的事实归因''，减少方向塌缩。

\subsection{为何第二轮进化没有自动晋升}

R2 表明``继续演化''不保证单调提高。候选在 train/dev 反馈上加入更严格的动量、换手、波动和持仓数量阈值，可能针对 2024--2025 的特定回撤形态过度补偿；holdout 市场状态变化后，防御规则没有继续占优。一次无效输出又会触发零容忍门禁。因此系统把 active skill 视为带版本的生产资产：新候选必须重新通过 dev 与封存集，而不是因为来自更晚一轮就覆盖旧版本。

\subsection{多智能体的代价}

多智能体试运行显示，筛选员与分析员能把职责和工具权限分开，但一月包含主智能体加两段子轨迹，平均耗时约为单智能体的数倍，也更容易触发供应商限频。架构收益必须覆盖额外调用成本和失败概率才有意义。由于完整 24 月对比尚未结束，本文不声称多智能体优于单智能体；这一负责任的留白本身也是实验纪律的一部分。

\subsection{效度威胁与局限性}
\label{sec:limitations}

\paragraph{样本与随机性。}
主对比只有 12 个月，Hermes dev/holdout 各 6 个月；每个 arm 只运行一次，GLM-5.2 在 temperature 1.0 下会产生显著路径差异。相同 active skill 在不同轮次的 holdout 总收益也有较大变化，说明单次 rollout 噪声不可忽略。未来应至少做多种子重复、bootstrap 置信区间和更长时间窗。

\paragraph{参数记忆与数据边界。}
行情、指数和估值查询按调用者给出的 \texttt{as\_of} 截断，财务表按公告日过滤，但这还不是形式化完备的点时数据仓。Plain fork 主要依靠题面禁止模型传入未来日期；24 月快照的 \texttt{stock\_basic\_all} 在 2026 年 7 月刷新，历史 ST 判断使用当前名称；财务缓存缺失时可能触发实时 Tushare 请求，且缓存对 2025 年 7 月后的新财报覆盖有限。基座模型参数也可能记住历史市场事件。禁止联网只能缓解这些风险，不能消除它们。复权与停牌处理同样只是回测近似，未建模涨跌停排队、盘口冲击、容量和滑点。

\paragraph{基线与风险调整。}
本文窗口恰逢全市场等权风格较强。虽然同时报告累计净收益、月频年化 Sharpe、最大回撤和费用，但 12 个或 6 个月观测不足以稳定估计风险调整绩效；取零无风险利率和使用月末净值回撤也只是统一比较约定。本文尚未进行跨市场状态的风险预算、因子暴露回归或现实容量分析。全市场等权参考不可交易，不能作为可实现收益基线。

\paragraph{归因不完全。}
主实验是两个 workspace fork 的完整系统对照，不仅联合打开 memory+skill，还存在点时守卫和组合修复差异；没有在同一代码路径、完全相同 rollout 噪声下完成全因子消融。R1 更严格地隔离了技能文本，但其候选由同一 train 反馈生成，仍可能过拟合。R2 失败和多智能体未完成结果限制了对``持续自进化''和``架构扩展''的结论。一次性 holdout 租约和晋升检查目前属于 CLI/registry 的应用层纪律，并非无法绕过的安全边界。

\paragraph{服务稳定性。}
API 限频可能在最后格式化阶段产生空 boxed。当前 evaluator 将其按现金处理，Hermes 门禁进一步一票否决；这保证结果不会被静默忽略，却把模型能力和基础设施可靠性共同计入端到端性能。正式生产评测应预先规定基础设施失败的重跑策略，并记录重跑预算。

\section{结论}
\label{sec:conclusion}

本文完成了从无状态工具智能体到金融长期认知系统的工程闭环：\memskill{} 在 \miroflow{} 上加入退出日禁运的事实型组合记忆和可路由程序性技能；开放池 benchmark 以冻结点时数据和确定性交易回放评价真实组合；Hermes 管线则把技能变更约束在可审计、可否决和可回滚的生命周期内。

12 个月主实验中，联合 memory+skill 系统相对 \miroflow{} 提升 $16.90\pp$ 累计净收益，月频年化 Sharpe 从 0.78 提高到 1.12，但承担更大回撤和成本。第一轮技能进化在 dev 与 holdout 都取得 5 胜 1 负并成功晋升，说明文本级程序知识可以在不训练模型参数的情况下被优化；第二轮失败则说明门禁和保守部署同样重要。后续工作应优先完成多种子重复和完整四臂消融，再评价多智能体是否以合理成本带来稳定增益。本文结果支持``长期认知与技能治理值得继续研究''，但不构成任何投资建议。

\section{开放池技能样例}
\label{sec:appendix-skill}

基础技能只定义流程，不定义答案。其核心伪代码如下：
\begin{quote}
\small
确认 \texttt{as\_of} 与交易约束；从成长质量、合理估值、高流动性和低波动等不同来源建立候选；批量交叉验证财务与估值；先剔除流动性不足、财务恶化和极端价格状态，再按证据一致性与行业互补排序；根据市场广度、候选质量和相关性决定 \cash{}；最后检查代码、权重、整手和输出格式。历史记忆只用于校准失误，不继承旧股票名单。
\end{quote}

R1 晋升版本在此基础上增加：广度过热时的现金下限、组合高动量暴露上限、行业/前三大持仓集中度约束、最低持仓数量，以及高价股整手预检。这些变化可直接由 candidate digest 和父版本 diff 审计。
